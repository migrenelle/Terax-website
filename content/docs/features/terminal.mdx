Terax features a custom terminal subsystem built for minimal latency, high throughput, and seamless integration with the host operating system. The subsystem splits execution between a high performance Rust backend and an optimized React frontend.

## Architecture Overview

Unlike traditional terminals that run entirely in a single process, Terax utilizes a two-process model for terminal execution:

1. **Rust Backend (portable-pty)**: Owns process lifecycle, pseudoterminal (PTY) spawning, and IO execution. The frontend webview does not communicate with shell processes directly. Instead, PTY events are streamed to the frontend over Tauri channels.
2. **React Frontend (xterm.js + WebGL)**: Receives PTY byte streams and renders them using a hardware-accelerated WebGL pipeline.

## Shell Spawning and Job Control

Terax ensures shell child processes are cleaned up when a session closes.

### Windows Process Lifecycle

Windows does not natively propagate close signals to child process trees. For example, if you spawn a development server inside PowerShell and close the PTY, the server process will remain orphaned.

To solve this, Terax assigns each Windows terminal session to a dedicated Job Object with the limit flag `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE`. When the terminal tab is closed or the host Terax process exits:

- The Job Object handle is released.
- The Windows kernel immediately terminates the shell process and all its descendants.

### Concurrent Spawning

Tauri handles parallel spawns via a global spawn mutex. This prevents Windows ConPTY output pipe locks during rapid tab spawning.

## Shell Integration Protocol

Terax injects custom scripts during shell initialization to track directory changes and command boundaries without parsing output.

### Unix Shells (Zsh / Bash)

Terax sets `ZDOTDIR` for Zsh or passes `--rcfile` for Bash to load injected scripts (`zshrc.zsh`, `bashrc.bash`). These scripts configure hooks to emit standard ANSI escape sequences:

- **OSC 7**: Emits the current working directory on directory changes.
- **OSC 133**: Emits markers for prompt starts, command starts, and exit codes.

### Windows (PowerShell)

A profile script (`profile.ps1`) is injected using the powershell execution flag. The script wraps the shell's active prompt function to output OSC 7 and OSC 133 sequences, converting forward slash paths to backslashes before transmitting.

## WebGL Renderer

The frontend terminal utilizes the WebGL addon of xterm.js. This renders the terminal grid directly on the GPU, avoiding standard HTML DOM rendering overhead. The terminal theme colors are synced directly from the central theme engine to the xterm color palette on load.
