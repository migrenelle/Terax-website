Terax stores application settings in a single JSON file. You can open and edit this file directly in any editor.

## File Locations

- **macOS / Linux**: `~/.config/terax/config.json`
- **Windows**: `%USERPROFILE%\.config\terax\config.json`

---

## Configuration Schema

```json
{
  "theme": "catppuccin-mocha",
  "font_family": "Geist Mono",
  "font_size": 13,
  "vim_mode": true,
  "terminal": {
    "renderer_type": "webgl"
  },
  "ai": {
    "provider": "openai",
    "model": "gpt-4o",
    "autocomplete": true
  }
}
```

### Root Fields

- **`theme`** (string): The color palette used across the workspace. Presets include:
  - `terax-default`
  - `nord`
  - `tide`
  - `catppuccin-mocha`
  - `tokyo-night`
  - `caffeine`
  - `claude`
  - `gruvbox-dark`
  - `rose-pine`
- **`font_family`** (string): The font family applied to the editor and terminal screens (e.g. `Geist Mono`, `JetBrains Mono`).
- **`font_size`** (number): Font size in pixels.
- **`vim_mode`** (boolean): Enables Vim keyboard input mode in editor tabs.

### Terminal Configuration (`terminal`)

- **`renderer_type`** (string): The renderer engine used by `xterm.js`. Options:
  - `webgl` (Recommended, hardware-accelerated rendering).
  - `canvas` (Fallback renderer).

### AI Configuration (`ai`)

- **`provider`** (string): The active LLM backend. Options:
  - `openai`
  - `anthropic`
  - `google`
  - `local` (LM Studio / Ollama)
- **`model`** (string): The model identifier used for general chat prompts (e.g., `claude-3-5-sonnet-latest`, `gpt-4o`).
- **`autocomplete`** (boolean): Enables context-aware inline code completions inside editor tabs.
