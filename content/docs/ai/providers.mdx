Terax operates on a Bring Your Own Key (BYOK) model. It does not charge usage premiums or lock you into a single LLM provider.

## Supported Cloud Providers

Terax connects to cloud AI providers using the Vercel AI SDK:

- **Anthropic**: Recommended for agent coding workflows (Claude 3.5 Sonnet).
- **OpenAI**: Supports GPT models (GPT-4o, o1-preview).
- **Google**: Gemini models.
- **Other Cloud Models**: Supports xAI, Cerebras, Groq, and any OpenAI-compatible custom base URL.

## Local & Offline Providers

For fully private, local development, Terax connects to local model providers:

- **LM Studio**: Connects via local servers.
- **Ollama**: Connects to local Ollama endpoints.
- **MLX**: Tailored for Apple Silicon execution.

Local models do not require active API keys. You specify the model identifier in your configuration file.

## API Key Security

Terax does not save API keys to local settings files, environment variables, or local storage.

Instead, keys are stored in the host operating system's secure keychain using the `keyring` crate in Rust. The keyring service is registered as `terax-ai`. On Linux environments without a running keychain service, Terax uses a secure file-based fallback system.
