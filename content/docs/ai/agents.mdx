Terax contains an agentic AI workflow engine that can write code, run command pipelines, and refactor codebases.

## The Composer Bar

The composer context acts as the input collector for agent interactions. You can attach multiple context segments before submitting prompts:

- **Files**: Attach active files to the prompt context.
- **Terminal Selections**: Drag or select terminal output to attach context as code blocks.
- **Voice Input**: Stream voice input to a transcription pipeline for speech-to-text entry.

## Agent Architecture

The agent runs as an execution cycle powered by Vercel AI SDK chat semantics:

- **The Main Agent**: Receives user prompts, reviews the workspace directory tree, and executes tool actions.
- **Sub-Agents**: The main agent can spawn specialized sub-agents with narrow prompt instructions and limited tool subsets (like code refactoring or directory search). Sub-agents report back to the parent agent upon completion.

## CLI Agent Integration

Terax detects CLI-based coding agents (like Claude Code) running inside PTY tabs. The Rust backend uses a byte-filter stream reader to identify prompt submissions and state transitions via OSC 133 sequences. This triggers visual state notifications in the header (working, attention, or finished) without interrupting terminal rendering.
